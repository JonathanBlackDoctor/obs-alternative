# SilentStream 사용자 안내서

OBS 없이 **PC를 켜면 자동으로 YouTube 라이브 송출 + 로컬 백업 녹화**가 시작되는
백그라운드 프로그램입니다. (Windows 10/11 x64 전용)

> ⚠️ **본인 소유 PC의 자기 기록·방송 목적 전용**입니다. 타인의 화면·음성을 동의
> 없이 캡처하는 용도로 사용할 수 없습니다 (EULA 참조).

## 1. 설치

1. `SilentStream-Setup-x.y.z.exe` 실행.
2. EULA(사용 동의) 확인 → 동의.
3. **자동 시작 방식** 선택:
   - **시작 프로그램(권장)** — 로그인 후 실행.
   - **작업 스케줄러** — 최고 권한, 로그인 직후 실행.
4. **녹화 폴더** 선택 (기본: `내 비디오\SilentStream`, 용량 큰 드라이브 권장).
5. 최초 실행 시 화면·오디오 캡처 동의 화면이 1회 표시됩니다.

### YouTube 연결 (최초 1회)
1. 최초 송출 시 브라우저가 열리면 **본인 Google 계정**으로 로그인.
2. 이후에는 자동 갱신되며 재로그인이 필요 없습니다.
3. 채널에 라이브 스트리밍이 활성화돼 있어야 합니다 (YouTube Studio → 라이브 스트리밍, 최초 활성화에 24시간 소요될 수 있음).

## 2. 일상 사용

- **PC를 켜면 끝** — 약 30초 후 자동으로 송출+녹화가 시작됩니다.
- 화면 좌측 상단의 **9px 점**이 상태를 보여줍니다:
  | 색 | 의미 |
  |---|---|
  | 🟢 | 정상 송출 중 |
  | 🟡 | 연결 중 |
  | 🔴 | 오류·재시도 중 (녹화는 계속됨) |
- 라이브는 항상 **일부 공개(Unlisted)** 로 생성됩니다. 링크를 아는 사람만 시청 가능.
- PC를 끄면 자동으로 방송이 종료되고 녹화 파일이 마감됩니다.

## 3. 제어판 (`Ctrl+Shift+F12`)

| 영역 | 기능 |
|---|---|
| 상태 배지 | LIVE / 연결 중 / 재시도 중 + 수동 시작·중지 |
| 성능 | 업로드 kbps, fps, CPU/GPU |
| 오디오 | 시스템·마이크 독립 볼륨, 마이크 장치 선택 |
| 자원 제한 | 25/50/75/제한 없음 (근사 상한, 다음 세션부터) |
| 녹화 | 현재 파일명, 누적 용량, 남은 디스크 |
| 로그 | 최근 1000줄 실시간 |
| 설정 | 단축키 변경, 자동 시작 방식, 녹화 폴더/한도/보존일 |

## 4. 녹화 파일

- 위치: 설치 시 지정한 폴더. 파일명 `SilentStream_REC_2026-06-12_0930.mp4`.
- 부팅~종료가 1개 파일입니다 (인코더 재시작 시 `_part2` 파일 추가).
- **7일**(설정 가능) 지난 파일과 **용량 한도**(기본 100GB) 초과분은 오래된 것부터 자동 삭제됩니다.
- 송출이 끊겨도 녹화는 계속됩니다. 갑자기 전원이 꺼져도 파일은 재생 가능합니다.

## 5. 문제 해결

| 증상 | 확인 사항 |
|---|---|
| 박스가 계속 🟡 | 인터넷 연결, YouTube 라이브 스트리밍 활성화 여부 |
| 박스가 계속 🔴 | `%AppData%\SilentStream\logs\` 최신 로그 확인. 자동으로 무한 재시도합니다 |
| 브라우저 로그인 안 뜸 | `%AppData%\SilentStream\client_secret.json` 존재 확인 |
| 마이크 소리 없음 | 제어판 → 오디오 → 장치 선택, 볼륨 슬라이더 확인. 마이크가 없어도 시스템 소리는 송출됩니다 |
| 녹화만 되고 송출 안 됨 | 정상 동작입니다(네트워크 문제 시). 복구되면 자동 재연결 |
| 게임 전체화면에서 검은 화면 | 자동 재초기화를 기다리거나 창 모드(테두리 없는 전체화면) 사용 |
| 단축키가 안 먹힘 | 다른 프로그램과 충돌 — 설정에서 다른 조합으로 변경 |

- 로그: `%AppData%\SilentStream\logs\SilentStream_날짜.log` (180일 보관)
- 설정: `%AppData%\SilentStream\config.json`

## 6. 교시 자동 분할 VOD 업로드

하루 종일 1개의 라이브는 그대로 유지하면서, **교시 시간표에 맞춰 녹화본을 교시별로 잘라
"N교시 - 날짜" 제목의 미등록(Unlisted) 영상으로 YouTube에 자동 업로드**합니다.
(쉬는 시간은 업로드되지 않습니다. 라이브/녹화 파이프라인은 전혀 건드리지 않고 **녹화본을 읽기만** 합니다.)

### 동작 방식
1. 교시 시간표를 입력합니다 (제어판이 아닌 **폰 원격**에서 입력 — §7, 또는 `config.json` 직접 편집).
2. 각 교시 종료 시각에 앱이 세션 녹화본에서 해당 구간을 **무손실(재인코딩 없음)** 로 잘라냅니다.
   - 컷은 가장 가까운 키프레임에서 시작하므로 시작점이 **최대 2초** 어긋날 수 있습니다(수업 녹화에 무해).
3. 잘린 파일은 **영속 업로드 큐**에 적재되어 순서대로 업로드됩니다.

### 시간표 입력 형식 (요일별 기본 + 그날만 덮어쓰기)
- **요일별 기본값**: 월~일 각 요일의 교시표를 한 번 정해두면 매주 반복됩니다.
- **오늘만 덮어쓰기**: 단축수업일 등 그날 하루만 다른 시간표로 교체할 수 있습니다(다음 날 자동 해제).
- 제목 형식: 기본 `"{교시}교시 - {yyyy-MM-dd}"` → 예 `1교시 - 2026-06-14`.

### 업로드 한도(quota)와 다음날 이월 — 중요
- YouTube 업로드 API는 **하루 약 6건**까지만 허용됩니다(`videos.insert` = 1,600 units, 일일 10,000).
- **7교시 이상**이면 그날 6건을 초과한 영상은 **큐에 보존**되었다가 **다음 날 자동 업로드**됩니다.
  - 큐는 `%AppData%\SilentStream\upload_queue.json` 에 영속 저장되어 **재부팅 후에도 유지**됩니다.
  - 리셋 시각: 태평양 시간(PT) 자정 = **한국시간 약 16~17시**. 이후 워커가 자동 재개합니다.
- 매일 모든 교시를 당일에 올리려면 **quota 증설 신청**을 권장합니다:
  Google Cloud Console → YouTube Data API → *Audit and Quota Extension* 양식(무료, 심사 수일~수주).
  승인 전에도 큐 덕분에 누락 없이 다음날 올라갑니다.
- 업로드가 라이브 업링크를 방해하지 않도록 기본값은 **즉시 + 대역 제한(immediate-throttled)** 입니다.
  학교 회선이 좁으면 `config.json` 의 `periods.uploadTiming` 을 `after-hours`(방과 후 일괄)로 바꾸세요.

## 7. 폰 원격 제어

같은 와이파이(또는 Tailscale)의 **폰 브라우저로 접속**해 교시 시간표 입력/수정, 라이브 시작·중지,
상태·업로드 진행을 확인합니다. **폰에 앱 설치가 필요 없습니다.**

### 켜기
- 설치 시 "폰 원격 제어 사용"을 체크하면 자동으로 켜집니다(LAN, 포트 8787, 방화벽 규칙 추가).
- 또는 `config.json` 의 `remote.mode` 를 `"lan"` 으로 바꾸고 앱을 재시작합니다.

### 연결(페어링)
1. 폰 브라우저에서 `http://<PC의 LAN IP>:8787` 접속 (PC IP는 제어판 로그 또는 `ipconfig` 로 확인).
2. **제어판(`Ctrl+Shift+F12`) → 설정 → 폰 원격 제어**에 표시된 **6자리 PIN**을 폰에 입력 → 연결.
3. 한 번 연결하면 폰에 **기기 토큰**이 저장되어 다음부터는 PIN 없이 접속됩니다.
   (토큰은 해시로만 PC에 저장되며 원문은 저장되지 않습니다.)

### 집 밖에서 (Tailscale 권장)
- LAN 모드는 같은 와이파이 전제의 평문(HTTP)입니다. **집 밖에서는 Tailscale** 사용을 권장합니다(트래픽 암호화).
  1. PC와 폰에 [Tailscale](https://tailscale.com) 설치·로그인(같은 계정).
  2. `config.json` 의 `remote.mode` 를 `"tailscale"` 로 변경 후 앱 재시작.
  3. 폰에서 `http://<PC의 Tailscale IP(100.x.x.x)>:8787` 로 접속.
- 공인 IP 직접 노출/포트포워딩은 보안상 권장하지 않습니다.

## 8. 설정 파일 추가 항목 (`config.json`)

```jsonc
{
  "periods": {
    "weekdayDefaults": { "Mon": [ { "no": 1, "start": "09:00:00", "end": "09:50:00" } ] },
    "overrides":       { "2026-06-15": [ { "no": 1, "start": "09:00:00", "end": "09:30:00" } ] },
    "titleTemplate": "{교시}교시 - {yyyy-MM-dd}",
    "vodPrivacy": "unlisted",
    "uploadTiming": "immediate-throttled"   // 또는 "after-hours"
  },
  "remote": { "mode": "off", "port": 8787, "deviceTokens": [ /* sha256 해시 */ ] }
}
```
- 보통은 **폰 원격 UI에서 시간표를 입력**하면 이 항목이 자동으로 채워집니다. 직접 편집도 가능합니다.

> 배포 참고: 폰 원격 서버는 ASP.NET Core 8 런타임을 사용합니다. 프레임워크 종속 배포 시
> 대상 PC에 **.NET Desktop Runtime 8 + ASP.NET Core Runtime 8** 이 필요합니다(인스톨러 번들 또는 자가 포함 배포로 해결).

## 9. 제거

제어판 → 앱 → SilentStream 제거. 자동 시작 등록·원격 방화벽 규칙은 함께 해제되며,
**녹화 파일과 설정은 삭제되지 않습니다** (직접 삭제 가능).
